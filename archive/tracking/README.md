# Tracking System

The `tracking/` folder is the live operational pulse for the project.

## Files

- `status.md`: current state and active focus.
- `decisions.md`: durable decisions and rationale.
- `risks.md`: known risks, severity, mitigation, and owner.
- `work-log.md`: chronological record of meaningful changes.
- `milestones.md`: larger product milestones and progress.
- `true-cert-factory.md`: focused tracker for the current platform unlock.

## Update Rules

- Update `status.md` whenever active focus, project phase, or verification state changes.
- Update `decisions.md` when the team makes a choice future agents should preserve.
- Update `risks.md` when a known risk appears, changes severity, is mitigated, or is closed.
- Update `work-log.md` for every meaningful foundation, architecture, implementation, release, or audit event.
- Update focused trackers when their related todo moves forward.

## Owner

The current active agent owns tracking updates during its work. The Steward tier is responsible for catching drift during audits.
